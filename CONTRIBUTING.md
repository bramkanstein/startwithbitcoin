# Contributing to Start With Bitcoin

Thank you for your interest in contributing! This guide will help you get started.

## Ways to Contribute

### 1. Write or Improve Guides
- Add new tutorials for specific use cases
- Improve existing guides with better explanations
- Fix errors in code examples
- Translate content to other languages

### 2. Add Code Examples
- Share working examples for common tasks
- Add examples for different programming languages
- Create complete project templates

### 3. Fix Issues
- Browse open issues on GitHub
- Submit pull requests with fixes
- Report bugs you find

### 4. Share Ideas
- Open issues to suggest new features
- Discuss improvements in existing issues
- Share how you're using Bitcoin with AI agents

## Getting Started

### 1. Fork and Clone

```bash
# Fork the repo on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/startwithbitcoin.git
cd startwithbitcoin
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Start Development Server

```bash
npm run dev
```

Visit [http://localhost:3000](http://localhost:3000)

### 4. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

### 5. Make Your Changes

Edit files, add content, or fix issues.

### 6. Test Your Changes

- Verify the site builds: `npm run build`
- Check for lint errors: `npm run lint`
- Test in browser manually

### 7. Commit Your Changes

```bash
git add .
git commit -m "Description of your changes"
```

### 8. Push and Create PR

```bash
git push origin feature/your-feature-name
```

Then open a Pull Request on GitHub.

## Guidelines

### Code Style
- Use TypeScript for all new code
- Follow existing patterns in the codebase
- Use Tailwind CSS for styling
- Keep components small and focused

### Content Style
- Write for developers new to Bitcoin/Nostr
- Use clear, simple language
- Include working code examples
- Test all code before submitting

### Commit Messages
- Use clear, descriptive messages
- Start with a verb (Add, Fix, Update, Remove)
- Keep the first line under 72 characters

### Pull Requests
- Fill out the PR template completely
- Link related issues
- Include screenshots for UI changes
- Keep PRs focused on one thing

## Code of Conduct

Be respectful and constructive in all interactions. We're all here to learn and build together.

## Questions?

Open an issue on GitHub. We're happy to help with your first contribution!
